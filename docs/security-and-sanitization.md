# Security And Sanitization

This repository is intentionally limited to a high-level case study. It is not a filtered mirror of the original work and should not be expanded with implementation artifacts unless written permission is obtained from the rights holder.

## Excluded From The Public Version

- Original git history.
- Source code, pseudocode, code snippets, and implementation samples.
- Database schema, SQL, migrations, functions, triggers, table names, and indexes.
- Workflow exports, workflow skeletons, node graphs, webhook paths, and automation configuration.
- Prompts, scoring logic, ranking logic, extraction rules, and source-specific behavior.
- Credentials, URLs, tokens, cloud identifiers, CRM identifiers, and deployment details.
- Product names, employer/client names, customer records, screenshots, financial records, and contact information.
- Runbooks, production logs, test results, build outputs, and original assistant files.

## Public Sharing Strategy

1. **Case study only:** describe the business problem, role, system areas, and lessons learned.
2. **No implementation artifacts:** avoid code, schema, query logic, raw workflows, prompts, or diagrams that reveal exact control flow.
3. **Anonymized language:** do not name the employer, product, customers, prospects, vendors, or internal systems unless the information is already approved for public use.
4. **Outcome-focused framing:** emphasize discovery, architecture, stakeholder communication, and delivery instead of proprietary mechanics.
5. **Interview-only depth:** discuss deeper tradeoffs live without publishing reusable implementation details.

## Ongoing Review Checklist

Before adding anything to this repo, check whether it reveals:

- How the production system works internally.
- A reusable version of employer-owned logic.
- Names, identifiers, screenshots, records, credentials, or private configuration.
- Client-specific requirements or business rules.
- Enough technical detail for someone to reconstruct the product.

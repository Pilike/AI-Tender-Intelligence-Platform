# Improvement Roadmap

This is a safe public portfolio backlog. It intentionally avoids suggesting publication of source code, schema, workflow exports, prompts, screenshots from production, or implementation details.

## Safe Public Improvements

1. **Add a one-page case-study PDF.** Summarize the business problem, responsibilities, technology categories, and outcomes without implementation detail.
2. **Add a public-safe diagram image.** Use the existing high-level diagram only; do not add exact infrastructure or workflow node maps.
3. **Add interview prompts.** Prepare private notes for how to discuss tradeoffs without revealing proprietary implementation.
4. **Add a generic lesson-learned section.** Focus on supportability, AI reviewability, and stakeholder communication.
5. **Create a separate personal demo later.** If a runnable demo is needed, build it from scratch with a different domain, different data model, different workflows, and no employer-derived logic.

## Do Not Add Publicly

- Raw cloud functions, service contracts, workflow exports, schema files, SQL, prompts, source-specific rules, or recreated implementation examples.
- Screenshots or videos from production systems.
- Any artifact that could be understood as a copy, derivative, or reusable version of the employer-associated product.
